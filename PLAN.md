# ПЛАН АНАЛИТИКИ ПЕРСОНАЖЕЙ TACTICUS

## 1. Подготовка структуры
- Создать файл `raw.md` — полная энциклопедия персонажей.
- Основной источник фактов: `data.json` из репозитория (брать статы, имя, скиллы).
- Генерация проводится строго по data.json (по алфавиту по `character.name`).
- Вести ALPHABET_PROGRESS — список всех обработанных и оставшихся героев.

## 2. Глубина анализа для каждого героя
### Из data.json брать:
- Имя, фракцию, статы (здоровье, броня, урон)
- Traits, пассивная и активная абилка, type/pierce атак

### Аналитика и описание:
- Роль героя (танк, дамагер, саппорт, киллер и т.д.)
- Краткое описание сильных и слабых сторон
- Лучшие команды/синергии (кратко и подробно)
- Лучшие/худшие соперники, оптимальные контры
- PvE/PvP рекомендации, рейды/арена/воро
- Советы по позиционированию и использованию
- Прочие особенности — FAQ, заметки

## 3. Алгоритм сбора и пуша
- Идти по алфавиту по именам character.name из data.json
- По каждому завершённому герою добавлять в ALPHABET_PROGRESS.md
- По окончании каждой десятки — пушить коммит на гитхаб с прогрессом
- Если герой пропущен — фиксировать причину (нет данных, баг)

## 4. Промежуточные результаты
- Промежуточные коммиты: каждая фракция или каждый десяток
- raw.md — накапливается по ходу работы
- ALPHABET_PROGRESS.md — список сделанного и to-do (чеклист)

## 5. Завершение
- Сводная таблица по героям
- Автоматизация импорта анализа в другие форматы (json/csv)

---
Старт работы фиксируется первым коммитом.
Точка остановки или паузы — по алфавиту и коммиту прогресса.
Все изменения описывать в коммитах на русском.
