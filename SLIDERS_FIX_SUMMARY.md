# ИСПРАВЛЕНИЯ ПОЛЗУНКОВ - ИНТЕГРАЦИЯ С DATA.JSON

## Основная проблема
Ранее ползунки редкости и уровня использовали абстрактные множители вместо реальных данных из data.json, что приводило к нереалистичным значениям характеристик.

## Ключевые изменения

### 1. Новая функция `getCharacterStatsFromData()`
- Извлекает реальные характеристики из таблиц активных способностей в data.json
- Поддерживает разные форматы таблиц:
  - `['Rarity', 'Level', 'Health', '3x Piercing']` (как у Abaddon)
  - `['Rarity', 'Level', 'Value']` (как у Angrax)
- Обрабатывает диапазоны значений (например "52-67" → среднее 60)
- Выполняет интерполяцию между уровнями
- Рассчитывает броню пропорционально HP

### 2. Обновление функции `renderAnalysisTab()`
- Автоматически устанавливает ползунки в соответствии с реальной редкостью персонажей
- Извлекает реальные уровни из таблиц для каждой редкости
- Больше не использует глобальные значения по умолчанию

### 3. Улучшенная функция `updateRarityCalculation()`
- Использует реальные данные из data.json вместо абстрактных множителей
- Показывает "(из data.json)" в заголовке блока характеристик
- Корректно обновляет все метрики расчета

### 4. Автоматический вызов обновления
- При открытии панели анализа автоматически вызывается `updateRarityCalculation()`
- Пользователь сразу видит реальные характеристики

## Преимущества
- Реалистичные значения характеристик из официальных данных
- Правильная интерполяция между уровнями
- Поддержка разных форматов таблиц
- Автоматическая инициализация ползунков
- Сохранение обратной совместимости (fallback к множителям)

Теперь ползунки работают с реальными данными из data.json!
